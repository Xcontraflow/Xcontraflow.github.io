25杭电多校暑期第四场 - 回忆与展望 1008

Author: contraflow Date: September 10, 2025  13:54:09 Category: 杭电多校

题面

Problem Description

通过梦境转换器，老师成功进入了编号最大的梦境。在那里，他遇到了 Seia。她的手上有 $n$ 份回忆，编号为 $1, 2, 3, \dots, n$。其中每一份都包含了老师与学生们度过的最美好的时光。这些日常时光交织在一起，共同组成了一个充满了无数奇迹的世界。
老师给编号为 $i \ (1 \leq i \leq n)$ 的回忆确定了它的 美好度 $a_i$。他想要按照某个顺序重温所有的回忆，也就是确定一个 $[1, n]$ 的排列 $p$ 并依次重温编号为 $p_1, p_2, p_3, \cdots, p_n$ 的回忆。

令 $a_0 = p_0 = 0$。老师对未来的 展望程度 可以用一个变量 $S$ 表示，初始时 $S := 0$。接下来在老师重温回忆的过程中，展望程度会根据美好度而发生变化。具体地，在老师重温编号为 $p_i \ (1 \leq i \leq n)$ 的回忆时：

  • 若 $a_{p_i} > a_{p_{i-1}}$，展望程度会增加 $x$，也就是 $S := S + x$。
  • 若 $a_{p_i} = a_{p_{i-1}}$，展望程度会增加 $y$，也就是 $S := S + y$。
  • 若 $a_{p_i} < a_{p_{i-1}}$，展望程度会增加 $z$，也就是 $S := S + z$。

Seia 通过某种方法得知了整数 $x, y, z$ 的值。她还知道老师喜欢美好，**所以 $x \geq y \geq z$**。她想要帮助老师确定排列 $p$，使得重温所有回忆后老师的展望程度最大。

Seia 警见了你，此时你或许正面对着一块电脑屏幕编写代码， 又或许正面对着一张草稿纸沉思。你或许正重温着以前年某次竞赛的试题，又或许对着一个极为重要的竞赛成绩感到懊悔。其实，她更建议你在竞赛结束后与付的同学们聊聊天，留意留意这个世界的某处角落，回忆回忆过去，展望展望未来。你可能在某一天突然会觉得，竞赛的具体知识和最终结果或许并不重要，而那份珍贵的回忆才是你人生中无法抹去的奇迹。
现在，是时候将这个问题告诉你了，Seia 想道。你只需要告诉她展望程度的最大值。

Input

本题包含多组测试数据。
来自 Seia 的温馨提示：本题输入输出量较大，对于使用 C++ 语言参加竞赛的选手，强烈建议使用关闭同步流的 cin 和 cout 完成输入输出。

首先在第一行输入一个整数 $T \ (1 \leq T \leq 100)$ 表示测试数据组数。
对于每一组测试数据，输入包含两行。

  • 第一行包含四个整数 $n, x, y, z \ (1 \leq n \leq \sum n \leq 5 \times 10^6,\ 1 \leq z \leq y \leq x \leq 5 \times 10^6)$，表示回忆的数量和计算展望程度所需要的三个参数。
  • 第二行包含 $n$ 个整数 $a_1, a_2, a_3, \dots, a_n \ (1 \leq a_1, a_2, a_3, \dots, a_n \leq n)$ 表示回忆的美好度。

Output

对于每一组测试数据，输出包含一行一个整数表示展望程度的最大值。

思路

这道题的本质，说白了就是让你对一个数组排序，怎么排，让数据尽量相等或递增，非必要不递减

可以想到的是，我们非必要尽量让相同的排在一起，总体保持递增
而不是一直递增然后错落有致的那种

贪心的证明就不给出了（实际上是本蒟蒻太懒），可以见后记

R[j] 从表示“有多少数恰好出现 ≥ j 次”，到“至少出现 j 次的数的个数”（前缀和）

Z 代表的是我们构造几个下降
Y 代表的是至少出现 i+2 个相同的个数
X 代表的是递增数

[!success]- 来自GPT的例子
例子：数组 [1,1,2,3,3]，一共有 n=5 个数。
第一步：统计每个数的出现次数 cnt
所以：cnt[1] = 2, cnt[2] = 1, cnt[3] = 2

第二步：计算 R
按定义 R[j] = 出现次数 ≥ j 的不同数的个数
R[1] = 3 （1,2,3 都至少出现 1 次）；R[2] = 2 （1,3 都至少出现 2 次）；R[3] = 0 （没有数出现 ≥3 次）；之后都是 0

第三步：枚举 i
i=0
此时 Z = i = 0，没有强行制造降序对；Y = R[i+2] = R[2] = 2，说明有 2 个数（1 和 3）能产生相等对；X = n - Y - Z = 3，剩下 3 个是升序对
i=1
此时 Z = i = 1，我们右移 1 位，前面腾出来的 1 个位置必然是降序对；Y = R[i+2] = R[3] = 0，说明没有数出现 ≥3 次，所以不能形成相等对；X = n - Y - Z = 4
i=2
此时 Z = 2；Y = R[4] = 0（没人出现 ≥4 次）；X = 5 - 0 - 2 = 3

所以 i 的含义就是：你让配对整体右移 i 位，于是保证拿到 i 个降序对。
然后看哪些数还能撑得住（出现次数 ≥ i+2），那些数就能提供相等对，其数量就是 R[i+2]。

+2 的原因是：
右移 i 位时，要让相同的值在配对里对上，至少需要前后两边各一个，加上间隔 i 个空位，所以一共要 i+2 个相同值

假设数值 7 出现了 t=3 次： [7,7,7]
如果 i=1：想配出相等对，需要至少 i+2=3 个 7，现在确实有 3 个，所以能成功
如果 i=2：需要至少 i+2=4 个 7，但只有 3 个，不够，所以对齐不出相等对
